# Cross-platform HTTP: mod list & changelog download on macOS/Linux

## Что сделано

- **Скачивание списка модов и чейнджлога** работает на macOS (и Linux), а не только на Windows: используется один и тот же libcurl, подтягиваемый через CMake FetchContent на всех платформах.

## Изменения

### CMakeLists.txt
- Убран `if (MSVC)` вокруг подключения curl: **curl собирается через FetchContent на всех платформах** (Windows, macOS, Linux).
- Один набор опций сборки curl для всех; только для `WIN32` оставлены Schannel и отключение OpenSSL.
- Вместо `HAVE_LIBCURL` вводится дефайн **`GAME_HAVE_CURL`**, чтобы не пересекаться с системным/внешним `HAVE_LIBCURL`.
- На non-MSVC добавлена линковка с `CURL::libcurl`, когда цель есть (раньше линковка была только при `find_package(CURL)` и `CURL_FOUND`, из-за чего на macOS линкер не находил символы curl).

### Код (mods.cpp, main_menu.cpp)
- Условия для HTTP-функций переведены на **`#ifdef GAME_HAVE_CURL`** (скачивание списка модов, скачивание мода, чейнджлог, GitHub API для коммитов).

## Результат

- На macOS (и при такой же сборке на Linux) в игре работают:
  - загрузка списка модов с репозитория;
  - скачивание модов;
  - загрузка чейнджлога с GitHub.
- Одна и та же библиотека (curl из FetchContent), без смены библиотеки на другую — только унификация сборки и линковки.
